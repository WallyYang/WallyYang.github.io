Configure Environment

Configure My Own Programming Environment

As a programmer, I love spending time to configure my own programming environment to help me become more efficient in programming.
Since most of the IDE (Integrated Development Environment) are provided with limited choice to customize, I chose emacs and Linux as my main working environment. So I kept maintaining a project to keep track with all the configuration code I made in my system in Github

Emacs is simply a text editor, so it does not ship with everything that an IDE provide, such as code completion. However, with certain customization, I can make it as good as an IDE. For example, I use C++ a lot in my projects, so I chose ycmd as my completion server.

As the picture shows, emacs can provide a good completion with the help of ycmd and some configuration codes I did. Meanwhile, since emacs is extremely customizable, itâ€™s easier to get along with some less-popular programming languages like haskell, scheme, rust, etc. And this makes emacs a really useful tool when I try to learning some new programming languages.

More importantly, IDEs are just coding tools, but I customize emacs with some codes so that it can help me a lot in my daily life.

I like using emacs to take notes in some self-learning process like reading new technology books. I customize emacs so that it can provide clear outline when I take notes from different chapters and sections.

David Allen is a famous author for writing and implementing the system to Getting Things Done (GTD). When I first read the book, I found emacs to be a perfect tool for me to put everything to organzie my works, both short-term ones and long-term ones.

With the customization of some codes in emacs, I am able to keep track of the works and projects and make reminders to notify me of things that is more urgent than the others.

There are many people who think spending too much time on programming tools is not good, because you could spend the time on actual project. But I believe tunning tools are important projects as well. The time I spent on emacs and Linux was not nothing. It helped me to do things more efficiently, and I actually learnt a lot when I tunning the tools. For instance, I learnt Bash scripts myself when I found myself needed a quick way to deploy my environment on a new machine, and I learnt the skills to manage source code when I study the codes in the open source community. Thus, I do recommend beginners to get their hands on some interesting codes to customize their system.
